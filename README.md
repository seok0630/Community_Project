# Community_Project
첫 프로젝트로 간단한 커뮤니티 어플리케이션 프로젝트입니다.  
코틀린과 파이어 베이스를 활용했습니다.   

## 개발도구 및 환경
+ 개발 도구: Android Studio
+ 프로그래밍 언어: 코틀린
+ 프레임워크: Firebase
+ 버전 관리 도구: GitHub

## 프로젝트 목적 및 배경
+ 2022년 겨울 방학을 맞이하여 대학교 2학년을 준비하는 과정에서 자바를 학습하였으나, 보다 심화된 학습과 개인 프로젝트를 진행하기에 자습만으로는 한계가 있다고 판단하였습니다. 이에 따라 좀 더 배우기 쉽고, 효율적인 코틀린으로 전환하여, 언어의 학습 뿐 아니라 실제 개발 경험을 쌓기위해 이 프로젝트를 시작하게 되었습니다.

+ 한국의 다양한 커뮤니티 어플리케이션을 참고하여, 사용자 친화적인 인터페이스와 기능을 구현하였습니다. 이를 통해 기존의 커뮤니티 어플리케이션들의 장점을 모아, 사용자들이 더욱 편리하게 소통할 수 있는 공간을 제공하고자 하였습니다.

## 프로젝트의 핵심 기능 소개
+ 이 커뮤니티 어플리케이션 프로젝트에서 핵심 기능으로 강조하고 싶은 부분은 구글 FireBase의 Cloud Firestore DataBase의 도입입니다.<br>
대부분의 기존 코틀린 관련 자료에서는 Realtime DataBase를 사용하여 커뮤니티 어플리케이션을 구현하지만, 이 프로젝트에서는 Cloud Firestore DataBase를 이용하였습니다.
Cloud Firestore DataBase는 Realtime DataBase의 상위호환격인 DataBase로 실시간 동기화를 강점으로 내세우는 NoSQL방식의 Realtime DataBase의 모든 장점을 유지한채 쿼리또한 Realtime DataBase 대비 아주 간단한 코드를 이용해 쓸 수 있다는 장점을 지닌 구글이 지원하는 최신 DataBase입니다.<br>
Cloud Firestore DataBase를 사용함으로써 글 정렬 등의 관리를 더욱 편리하게 할 수 있게 되었으며, 이렇게 프로젝트에서 구글이 권장하는 차별화된 기술을 도입함으로써 사용자에게 보다 안정적이고 편리한 사용 경험을 제공하고자 하였습니다.

## 프로젝트 평가 및 개선 방향

### 장점
+ 전체적인 디자인이 이미 시장에서 검증된 UI를 참고하여 구현하였기에 사용자에게 친숙할 것이며, 불편한 위치에 있는 요소가 적습니다.
+ Cloud Firestore를 사용하여 다양한 쿼리를 활용할 수 있으며, 이를 통해 Realtime Database대비 난잡한 코드를 줄이고 사용자의 데이터를 유지보수하기에 용이합니다.

### 단점
+ 이미지를 원본 그대로 스토리지에 저장하는 방식으로, 앱이 압축을 진행하지 못해 대량의 트래픽이 발생합니다. (20mb가 넘어가는 Gif형식의 이미지가 8명의 사용자에게 보여지자 대량의 트래픽이 발생한 일이 있었음)
+ 완성도가 높지 않아 일부 기능이 겉으로만 구현된 경우가 있습니다 (예: 글쓰기 중 비디오버튼이나 글씨 크기 조절버튼)

### 개선방향
+ 스토리지에 업로드할 수 있는 이미지 크기를 제한한다.
+ 추가적인 업데이트를 통해 완성도를 높이고 기능을 개선한다.

## 결론
+ 이 프로젝트는 코틀린을 학습한 결과를 확인하고, 그 실력으로 커뮤니티 어플리케이션을 얼마나 구현할 수 있는지를 평가하는 좋은 기회였습니다.
또, 프로젝트를 진행하며, 코틀린과 데이터베이스 관리 능력이 그 전보다 많이 향상되었다는 것을 느꼈습니다.

+ 데이터베이스를 Cloud Firestore로 전환하는 과정에서 스스로 문제를 인식하고 해결한 경험은 이후 프로젝트에서도 도움이 될 것이라고 생각합니다. 또, 다음 개인 프로젝트에서는 이번 프로젝트의 경험을 바탕으로 더 높은 완성도와 효율성을 지닌 어플을 만들어 낼 것이라고 자신 스스로 기대하게 되는 프로젝트 였습니다.
